Personal Cybersecurity:


Browser Tips for Websites Requiring Personal Data (e.g. banks)

  1. Ensure URL at top has "https://" and lock symbol.
  2. Ensure domain is not misspelled, e.g. avoid http://bank0famerica.com.
  3. Have only one browser tab open (a malicious website in another tab may do harm).
  4. Close browser after completing online transaction (stops any running scripts, etc.).

Email Safety

  1. Note that the "From" email address is easily faked: don't trust it.
  2. Note that link text can be different from where the link actually leads.
  3. Hover mouse over link and look at bottom left of browser to see where link leads.
  4. If email looks suspicious, delete it (even opening an email may verify to sender that the email reached you: e.g. when browser downloads linked images).
  5. Don't risk downloading or opening suspicious attachments.
  6. If you must open an attachment, then select a known program instead of double-clicking: right-click -> open with -> notepad, pdf reader, word, etc.
  7. When replying, check that all To, CC, and BCC addresses are legitimate.
  8. Slighly more advanced is to check email headers. In Gmail: open email -> click three dots (top right) -> "Show Original."
  9. Search for the sender domain, e.g. aol.com for john@aol.com, which should be throughout the headers: return path, auth results, etc.
  10. The headers check does not help much for cloud-hosted accounts, but can still show hints: e.g. suspicious email path.

Passwords

  1. Change regularly (make a calendar reminder, e.g. for once a year).
  2. Change if you see anything suspicious on your account or if you accidentally share any part of it.
  3. Change default passwords: e.g. routers and other devices use default passwords that anyone can find online.

Backups

  1. Either automate backups or make a calendar reminder to backup to two or three places: flash drives, external disks, etc.
  2. Systems crash, disks get dropped and damanaged, hackers encrypt drives (ransomware). It's good to have a backup.

Encrypt Drives

  1. Encrypt any medium that stores confidential data. Many new operating systems do this by default.
  2. Encrypt removable media. If you lose a laptop or flash drive, others will not see your data.
  3. It's best not to put confidential data into the cloud, but if need be, encrypt files.
  4. As of this writing Google drive doesn't have an encrypt feature, so encrypt locally prior to uploading to cloud.
  5. E.g. LibreOffice Writer (roughly open source version of MS Word): File -> Properties -> Security -> Protect.

Air Gap

  1. If you are not using the internet for an extended period: disconnect. E.g. disable network or default route.
  2. If your are not connected to the internet then hackers cannot attack you from afar nor can you leak confidential data.

General Idea:

Check that destination of outgoing information is legitimate: URL you are going to, phone you are calling, email you are sending data to, bank account you are depositing into, etc.
